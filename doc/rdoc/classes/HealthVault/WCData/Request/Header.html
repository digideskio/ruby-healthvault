info_hash=(value)

summary: The hash of the info section. remarks: This hash ensures that the info section is not tampered with. If the request is not authenticated, then this section is not required or expected. That is, in the authenticated request case, the header is protected via the HMAC operation over the header element, thereby preventing tampering. If the header is not protected then the info-hash can be modified arbitrarily. Microsoft HealthVault will compute the corresponding hash and compare the results. If they are not the same, the request will fail. value is a HealthVault::WCData::Types::HashFinalized
